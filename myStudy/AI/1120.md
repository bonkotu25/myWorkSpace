#### APIキー直接利用 vs 統合ツール（Cursor/Copilot）

- APIキー直接利用
  - GitHub Copilot、OpenAI Codex、Claude Codeの3つはそれぞれ自前のLLMモデルを持っており、それをAPIキー経由で呼び出すことが可能です。
- 統合ツールのLLM利用
  - 統合ツールは自前でLLMモデルを持っていない代わりに使用するLLMを選択することが可能(claude sonetでは解決できない問題をGPT-5.xに切り替えて解決する。)
  - GitHubへのプッシュのプッシュをトリガーに、AIに一部資材を自動生成させてプッシュ+ リリースさせる。

#### 結論

**※個人の感想です。**

- LLM切り替えで解決できるコアな課題よりも、APIキー経由のアクセスを使用できることで広がる選択肢のほうが幅広い
  - 例：異常発生時、サーバーから何らかのトリガーでAIを呼び出し一時解析をやったうえで通知させる。(exceptionをキーにして頭数行をAPIに渡してAIはクラスやexceptionから何が起こったかを判断し回答する。)

#### Claude codeの機能まとめ

[参考](https://zenn.dev/flinters_blog/articles/711ff6faeca62c)


- CLAUDE.md(全体ルール)
  - cursorでいうところのプロジェクトルール
- Slash Commands
  - ユーザーが独自のコマンドを作れる。例：レビュー用コマンドなど
- Sub-Agents
  - コンテキストウィンドウを分離させて別タスクとして作業をさせる。
- Hooks
  - イベントハンドラ。イベント駆動なので確実に起動させることができる。プロンプトに問わず確実に実行させることができるのでログ出しなどに使える。
- Plugins
  - ユーザーが自作したスラッシュコマンドなどを共有することができる。チームの環境統一などに使用できる。
- Agent Skills
  - AIで判断して必要な部分だけを読み込むことができる。前提知識や設定などでコンテキストが溢れないようにできる。
